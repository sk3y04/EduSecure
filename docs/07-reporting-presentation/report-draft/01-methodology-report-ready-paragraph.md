# Report-Ready Methodology Paragraph

## Paste-ready paragraph

EduSecure evaluates application security using a combination of complementary and widely recognised methodologies rather than relying on a single checklist. OWASP provides the main web-application security lens by highlighting common risk areas such as broken access control, cryptographic failure, insecure configuration, and weak monitoring, while OWASP ASVS and WSTG give more structured verification and testing guidance for authentication, session handling, authorization, file-upload security, and browser-facing controls. These application-focused methods are supported by a simplified NIST SP 800-30-style risk assessment to identify assets, vulnerabilities, threat events, likelihood, impact, and residual risk, and by CVSS v3.1 scoring to prioritise the most serious technical weaknesses. In addition, STRIDE strengthens the design discussion by framing threats such as spoofing, tampering, repudiation, information disclosure, denial of service, and privilege escalation across the platform architecture, while CIS-aligned hardening principles help connect the report to secure deployment concerns such as secret management, TLS posture, container configuration, and least-privilege operation.

## Use note

Use this paragraph in the methodology or risk-introduction part of the report, then follow it with a short table showing how each framework contributes a different analytical role.
